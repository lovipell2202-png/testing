ISSUE:
═════════════════════════════════════════════════════════════════════════════

Browser was caching old versions of JavaScript files that had duplicate
formatDate declarations, even though the files were fixed.

SOLUTION:
═════════════════════════════════════════════════════════════════════════════

✅ Added version query parameters to all script tags
   • ?v=2 forces browser to reload files instead of using cache

✅ Removed ALL duplicate formatDate declarations:
   • Removed from all-employees.js
   • Removed from training-records.js
   • Kept only in ui-helpers.js (single source of truth)

✅ Updated all formatDate() calls to use window.UIHelpers.formatDate()

WHAT THE VERSION PARAMETER DOES:
═════════════════════════════════════════════════════════════════════════════

OLD: <script src="src/all-employees.js"></script>
     → Browser caches this file

NEW: <script src="src/all-employees.js?v=2"></script>
     → Browser treats this as a NEW file and reloads it
     → Cache is bypassed
     → Latest version is loaded
